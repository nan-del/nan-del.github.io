蜜罐学习（一）

Posted on 2021-06-07 Edited on 2021-06-25

蜜罐的分类：

  • 部署目标

    • 产品型：就类似于成品服务，可以直接供用户使用，不用专业的无人员分析样本，能够把攻击流量吸引走，防止攻击真实的目标
    • 研究性： 可以近距离观察样本，对样本进行解构，一般用于研究攻击者的行为和发现新的攻击方式

  • 交互型：攻击者在蜜罐中活动的交互性级别

    • 低交互型：容易通过和蜜罐的交互中，辨别出这是否是一个蜜罐，还是一个真的系统，如一些基本的命令都执行不了，就容易被攻击者分辨出来这是个蜜罐
    • 高交互型：交互性高，可以使得黑客更加容易相信他攻陷的是一个真正的系统，而不是一个蜜罐，可以以假乱真，但是仿真程度越高，也就意味着暴露的资产也就越多，反而可能会被黑客利用，作为跳板，扫描内网，也可能通过虚拟机的相关漏洞拿到宿主机的权限
    • 混合型：就是交互型较高，但是给到的权限不会太高

  • 新型蜜罐

    • 主动式蜜罐
    • 蜜场
    • 蜜信
    • 应用层蜜罐
    • 客户端蜜罐

    产品型蜜罐

    • 部署目标：保护单位网络
      • 防御
      • 检测
      • 帮助对攻击的响应
    • 可以使得网管尽可能少的工作
    • 商业产品
      • KESenssor

    研究型蜜罐

    • 部署目标：对黑客攻击进行捕获和分析
      • 了解黑客在干什么
      • 了解攻击方法
      • 捕获他们的击键记录
      • 捕获他们的攻击工具
      • 监听他们的会话
    • 需要大量的时间和精力投入
    • 实例：
      • Genll蜜网
      • Honeyed

    低交互蜜罐

    • 模拟服务和操作系统
    • 只能捕获少量信息
    • 容易部署，减少风险
    • 实例
      • Dionaea
      • Honeyd
      • KFSensor

    高交互蜜罐

    • 提供真实的操作系统
    • 可以捕获更丰富的信息
    • 部署复杂，高安全，高风险
    • 实例
      • ShadowDaemon
      • Lyrebird

    主动式蜜罐技术

    • 动态蜜罐
      • 即插即用的解决方案
      • 自动调整
    • 被动指纹识别技术
      • 了解所处的网络环境
    • 动态配置
      • 虚拟蜜罐

    蜜场的基本思想

    • 把一堆蜜罐放进一个比较大的网络环境中，扮演一个有很多漏洞的大型网络，吸引攻击者，吸引攻击流量

    • 当内网中检测到了扫描行为，就把流量重定向到蜜场，蜜场中包含有很多蜜罐，就会把目标转移到蜜场中的虚拟的系统中，而遗漏真实的系统，使得攻击者把更多的时间和精力放在虚拟的蜜场中

    重定向机制

    • 对高价值目标映射蜜罐系统
      • 动态蜜罐技术
    • 不需要创建新的目标，而使用已存在的目标
    • 将恶意的、未经授权的活动重定向到蜜罐
    • 监视和捕获攻击者在蜜罐中的活动
    • 计算机取证技术

    什么时候重定向

    • 非预期的流量-Hot Zoning
      • 非业务目标端口
      • 非业务源端口
      • Time of day
    • 已知攻击-Bait-n-Switch
      • Snort的修改版本，内联网关
      • 将检测到的活动重定向到蜜罐中
    • 基于主机的监控
      • 监控主机上的未授权活动、或者恶意活动，然后重定向
      • 实例
        • PaX
        • Systrace

    蜜罐的欺骗性

    • 真实的系统环境
      • 系统标识
        • IP、traceroute路径
        • 主机名、操作系统内核版本等等细节
    • 系统配置和应用程序
      • 开放的网络服务
      • 安装的应用程序，假如一个攻击者已经攻陷了一台内网的主机，但是这个主机上什么应用程序都没有，就容易使人怀疑，因为正常的有人使用的电脑，普通的应用程序还是有的
    • 数据内容
      • 如一些文本数据，假如攻击者已经攻陷了一台主机，但是这台主机上什么数据都没有，连一个txt文本都没有，就容易使人怀疑
      • 数据内容尽量真实可信，但又不是真正的数据，让攻击者分辨不出，以为获得了敏感数据
      • 还有一些网页的历史浏览记录等，也应模拟出来
    • 优势：在一个高度可控的环境中愚弄/观察攻击者的所有行动

    蜜信（HoneyToken）

    • 概念出发点：威胁不仅仅是针对信息系统，也针对信息本身
    • HoneyToken:正常情况不会使用的一些诱饵信息
      • 数据库诱饵记录
      • 伪造的弱用户名、弱口令
    • 如果HoneyToken一旦被访问，预示着攻击即将发生，就可以对它进行监视跟踪

    应用层蜜罐

    • 模拟应用层的服务对非自动化的攻击更具吸引力
    • 在应用层能够更有效地对攻击进行分析
    • 实例
      • Web application honeypot

    客户端蜜罐（Honeyclient）

    • 越来越多的攻击针对客户端软件
      • Web浏览器
      • Email客户端
    • Honeyclient的设计要考虑
      • 应用协议本身
      • 需要捕获的攻击
    • 基于完整性测试判断是否遭受攻击
      • 程序本身
      • 配置文件、注册表等
    • 分类
      • Active honeyclient:同步交互，如web-based honeyclient
      • Passive honeyclient:异步交互，如wmail honeyclient

    蜜罐技术优势

    • 高保真–高质量的小数据集
      • 低误报率和低漏报率
    • 捕获新的攻击及战术
    • 并不是资源密集型

    蜜罐技术弱势

    • 劳力/技术密集型
    • 局限的视图
    • 不能直接防护信息系统
    • 引入新的安全风险
      • 如蜜罐被攻陷

    安全风险

    • 发现蜜罐
      • 蜜罐指纹识别 VS 消除蜜罐的指纹
        • 一旦识破蜜罐：攻击者可能会向蜜罐反馈虚假、伪造的信息
    • 利用蜜罐攻击的第三方
      • 期望黑客获得蜜罐的root权限 VS 获得root权限之后是否真的可以为所欲为
        • 系统仿真程度越高、开放服务越多会引入更多的攻击暴露面
      • 完全隔离网络 VS 伴随仿真蜜网
